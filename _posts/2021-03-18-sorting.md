이 페이지에서는 **정렬 문제**를 푸는 다양한 알고리즘을 제시한다. 정렬 문제는 다음과 같이 정의한다.

- 입력: 숫자 $$n$$개로 이루어진 수열 $$\langle a^\prime_1, a^\prime_2, ..., a^\prime_n \rangle$$
- 출력: 입력 수열 $$\langle a^\prime_1, a^\prime_2, ..., a^\prime_n \rangle$$을 $$a^\prime_1 \leq a^\prime_2 \leq ... \leq a^\prime_n$$ 순서로 재배치한 수열

입력은 보통 원소 수가 $$n$$개인 배열로 주어지지만, 연결 리스트와 같이 다른 형태로 주어지기도 한다.

파이썬과 같은 실제 프로그래밍 언어로 구현한 코드에서만 인덱스가 0부터 시작하는 zero-based 배열을 사용할 것이고 그 외 알고리즘을 설명할 때는 1-based 배열을 사용할 것이다.

# 삽입 정렬 (Insertion Sort) <a name="insertion-sort">

**삽입 정렬**은 아마 인간에게 가장 직관적인 정렬 방법일 것이다. 카드놀이를 할 때 손에 쥔 카드를 정렬하는 것과 방법이 같다. 이 알고리즘은 배열의 두 번째 요소부터 마지막 요소까지 순회하면서 현재 가리키고 있는 $$n$$번째 요소에 대해 1 ~ $$n-1$$번째 요소를 역순으로 순회하고, $$n$$번째 요소보다 작은 요소가 발견되는 즉시 그 자리에 **삽입**한다.

![/assets/images/sort/insertionSort.gif](/assets/images/sort/insertionSort.gif)

## 구현 <a name="insertion-sort-implementation">

### 의사코드

![/assets/images/sort/U14ab.png](/assets/images/sort/U14ab.png)

### Python

```python
def insertion_sort(A):
    n = len(A)
    for j in range(1, n):
        key = A[j]
        i = j - 1
        while i >= 0 and A[i] > key:
            A[i + 1] = A[i]
            i = i - 1
        A[i + 1] = key
```

## 성능 <a name="insertion-sort-performance">

배열을 순회하면서 처음에는 두 번째 요소부터 접근하고 비교 대상은 첫 번째 요소 하나만 존재한다. 그 다음에는 세 번째 요소에 접근하고 비교 대상은 두 번째 요소와 첫 번째 요소가 된다. 이 과정을 반복하면 비교 횟수는 다음과 같다.

$$1 + 2 + 3 + ... + (n-2) + (n-1) = \sum_{i=1}^{n-1}i$$

이는 등차수열의 합 공식($$S_n = \frac{n(a_1 + a_n)}{2}$$)에 의해 다음과 같이 표현될 수 있다.

$$\sum_{i=1}^{n-1}i = \frac{1+(n-1)}{2}(n-1) = \frac{1}{2}n(n-1) = \frac{1}{2}(n^2-n)$$

따라서 이 알고리즘은 요소를 비교하는 데에 있어 $$\Theta(n^2)$$의 수행시간이 소요된다. 만약 배열이 이미 정렬되어 있는 경우, 비교를 생략하기 때문에 이때는 최선의 경우가 되며 $$\Theta(n)$$의 수행시간이 소요된다.

# 선택 정렬 (Selection Sort) <a name="selection-sort">

![/assets/images/sort/selectionSort.gif](/assets/images/sort/selectionSort.gif)

**선택 정렬**은 배열의 첫 번째 요소부터 마지막으로부터 두 번째 요소까지 순회하면서 $$n$$번째 이후의 최소값을 가지는 요소를 **선택**한 뒤 $$n$$번째 요소와 위치를 바꿈으로써 배열을 정렬하는 알고리즘이다.

## 구현 <a name="selection-sort-implementation">

### 의사코드

```
SELECTION-SORT(A)
1   for i = 1 to A.length - 1
2       min = i
3       for j = i + 1 to A.length
4           if A[j] < A[min]
5               min = j
6       exchange A[i] with A[min]
```

### Python

```python
def selection_sort(A):
    for i in range(len(A) - 1):
        min_ = i
        for j in range(i + 1, len(A)):
            if A[j] < A[min_]:
                min_ = j
        A[i], A[min_] = A[min_], A[i]
```

## 성능 <a name="selection-sort-performance">

처음에는 먼저 $$n$$개의 요소를 스캔하고 나머지 $$n-1$$개의 요소와 비교한다. 그 다음에는 $$n-1$$개의 요소를 스캔하고 나머지 $$n-2$$개의 요소와 비교한다. 이런 방식으로 계속 진행하면 총 비교 횟수는 다음과 같다.

$$(n-1) + (n-2) + ... + 1 = \sum_{i=1}^{n-1}i$$

이는 등차수열의 합 공식($$S_n = \frac{n(a_1 + a_n)}{2}$$)에 의해 다음과 같이 표현될 수 있다.

$$\sum_{i=1}^{n-1}i = \frac{(n-1)+1}{2}(n-1) = \frac{1}{2}n(n-1) = \frac{1}{2}(n^2-n)$$

이는 삽입 정렬과 마찬가지로 입력 $n$개에 대해 $$\Theta(n^2)$$의 수행시간을 가진다. 배열이 어떻게 나열되어 있던 간에 모든 원소는 항상 비교되어야 하므로 최선의 경우나 최악의 경우나 비슷한 성능을 보인다.

# 병합 정렬 (Merge Sort) <a name="merge-sort">

잠깐 다른 주제에 대해 이야기하자면, 어떤 알고리즘을 설계하는 방법에는 여러 가지가 있다. 삽입 정렬은 **점진적**인 방법을 사용한다. 즉, 원소 $$A[1..j]$$를 정렬된 부분 배열 $$A[1..j-1]$$의 적절한 위치에 삽입하여 다시 정렬된 부분 배열 $$A[1..j]$$을 만든다.

그리고 **분할정복**이라고 알려진 또 다른 방법론이 존재한다. 이 방법은 전체 문제를 원래 문제와 유사하지만 크기가 작은 몇 개의 부분 문제로 분할하고, 부분 문제를 **재귀적**으로 푼다. 분할정복은 재귀 호출을 하는 각 단에서 다음 세 가지 단계로 구성된다.

분할: 현재의 문제를 같은 문제를 다루는 다수의 부분 문제로 분할한다.

- 정복: 부분 문제를 재귀적으로 풀어서 정복한다. 부분 문제의 크기가 충분히 작으면 직접적인 방법으로 푼다.
- 부분 문제의 해를 결합하여 원래 문제의 해가 되도록 만든다.

**병합 정렬**은 분할정복 기법을 아주 잘 이용한다. 직관적으로 설명하면 다음과 같다.

- 분할: 정렬할 $$n$$개 원소의 배열을 $$n/2$$개씩 부분 수열 두 개로 분할한다.
- 정복: 병합 정렬을 이용해 두 부분 배열을 재귀적으로 정렬한다.
- 결합: 정렬된 두 개의 부분 배열을 병합해 정렬된 배열 하나로 만든다.

## 구현 <a name="merge-sort-implementation">

병합을 하기 위해서는 보조 프로시저 `MERGE(A, p, q, r)` 이 필요한데, 여기서 $$A$$는 배열이고 $$p, q, r$$은 인덱스로 $$p \le q \lt r$$을 만족한다. 즉, $$p$$는 부분 배열 맨 왼쪽의 인덱스, $$r$$은 부분 배열 맨 오른쪽의 인덱스, $$q$$는 배열을 두 부분으로 나누기 위한 정가운데 인덱스를 나타낸다. 이 프로시저는 두 부분 배열 $$A[p..q]$$와 $$A[p+1..r]$$이 정렬되어 있다고 가정하고 이를 **병합**해서 정렬된 배열 하나를 만드는데, 이것이 원래의 부분 배열 $$A[p..r]$$을 대체한다.

### 의사코드

![/assets/images/sort/mergesort.png](/assets/images/sort/mergesort.png)
![/assets/images/sort/merge.png](/assets/images/sort/merge.png)

### Python

```python
def merge_sort(A, p, r):
    if p < r:
        q = (p + r) // 2
        merge_sort(A, p, q)
        merge_sort(A, q + 1, r)
        merge(A, p, q, r)

def merge(A, p, q, r):
    n1 = q - p + 1
    n2 = r - q
    L = [0] * (n1 + 1)
    R = [0] * (n2 + 1)
    for i in range(0, n1):
        L[i] = A[p + i - 1]
    for j in range(0, n2):
        R[j] = A[q + j]
    L[n1] = float("inf")
    R[n2] = float("inf")
    i = 0
    j = 0
    for k in range(p - 1, r):
        if L[i] <= R[j]:
            A[k] = L[i]
            i += 1
        else:
            A[k] = R[j]
            j += 1
```

## 성능 <a name="merge-sort-performance">

$$c$$를 하나의 문제를 푸는 데 드는 비용을 나타내는 상수라고 하고, $$n$$은 문제의 개수라고 하면 병합 정렬의 재귀 트리를 다음과 같이 나타낼 수 있다.

![/assets/images/sort/recurrence-tree-mergesort-3.jpg](/assets/images/sort/recurrence-tree-mergesort-3.jpg)

그러면 이 트리의 높이는 $$\lg n + 1$$이기 때문에 총 비용은 $$cn \times (\lg n + 1) = cn \lg n + cn$$이 된다. 여기서 상수 $$c$$와 낮은 차수의 항들은 무시할 수 있기 때문에 $$O(n \lg n)$$으로 나타낼 수 있다.

# 버블 정렬 (Bubble Sort) <a name="bubble-sort">

![/assets/images/sort/bubble-sort.gif](/assets/images/sort/bubble-sort.gif)

**버블 정렬**은 비효율적이지만 인기 있는 정렬 알고리즘이다. 이는 정렬되지 않은 이웃 원소를 반복적으로 바꿔 수열을 정렬한다.

## 구현 <a name="bubble-sort-implementation">

### 의사코드

```
BUBBLESORT(A)
1   for i = 1 to A.length - 1
2       for j = A.length downto i + 1
3           if A[j] < A[j - 1] then
4               swap A[j], A[j - 1]
```

### Python

```python
def bubble_sort(A):
    for i in range(0, len(A) - 1):
        for j in range(len(A) - 1, i, -1):
            if A[j] < A[j - 1]:
                A[j], A[j - 1] = A[j - 1], A[j]
```

## 성능 <a name="merge-sort-performance">

- 최악의 경우: $$\Theta(n^2)$$
→ 배열이 역순으로 정렬되어 있는 경우 모든 요소에 대해 나머지 모든 요소를 확인해야 한다.
- 최선의 경우: $$\Theta(n)$$
→ 배열이 이미 정렬되어 있는 경우 한 요소에 대해 나머지 요소를 확인할 필요 없이 다음 요소로 넘어가면 되기 때문에 안쪽의 루프는 시작되지 못하고 바깥의 루프만 돌게 된다.

# 힙 정렬 (Heap Sort) <a name="heap-sort">

병합 정렬처럼 (삽입 정렬과는 다르게) 힙 정렬의 수행시간은 $$O(n \lg n)$$이다. 그리고 삽입 정렬처럼 (병합 정렬과는 다르게) 힙 정렬은 **내부 정렬**이다. 즉, 정렬할 때 입력 배열에서 상수 개의 원소를 초과해서 배열 밖에 저장하는 일은 없다. 따라서 힙 정렬은 이미 논의한 두 정렬 알고리즘의 장점을 혼합한 것이다.

힙 정렬에서는 또 다른 알고리즘 설계 기술을 도입한다. 알고리즘 실행 중 정보를 다루기 위해 특수한 자료 구조인 "힙"이라는 자료구조를 이용한다. 힙 자료구조는 힙 정렬에 사용할 뿐만 아니라 효율적인 우선순위 큐를 만드는 데도 이용할 수 있다.

"힙"이라는 용어는 원래 힙 정렬로부터 나온 말이다. 그러나 이 용어는 Lisp나 Java와 같은 프로그래밍 언어가 제공하는 "공간"이라는 뜻으로 사용되어 왔다. 그러나 여기서 힙 자료구조는 공간이 아니며, 특수한 규칙을 가지는 트리 형태로 데이터를 처리하는 자료구조를 의미한다.

![/assets/images/sort/03DAF88F-A933-4AAE-BE1E-476B1EAC2754.gif](/assets/images/sort/03DAF88F-A933-4AAE-BE1E-476B1EAC2754.gif)

## 구현 <a name="heap-sort-implementation">

### 의사코드

```
HEAPSORT(A)
1   BUILD-MAX-HEAP(A)
2   for i = A.length downto 2
3       exchange A[1] with A[i]
4       A.heap-size = A.heap-size - 1
5       MAX-HEAPIFY(A, 1)
```

### Python

```python
from heapq import _heapify_max as max_heapify

def heap_sort(A):
    max_heapify(A)
    for i in range(len(A), 1, -1):
        A[0], A[i-1] = A[i-1], A[0]
        X = A[:i-1]
        max_heapify(X)
        A[:i-1] = X
```

## 성능 <a name="heap-sort-performance">

MAX-HEAPIFY 연산을 한 번 호출할 때마다 $$\Theta(\lg n)$$ 시간이 걸리고, 호출 횟수는 $$\Theta(n)$$번이기 때문에 수행 시간은 $$\Theta(n \lg n)$$이다.

# 퀵 정렬 (Quick Sort) <a name="quick-sort">

퀵 정렬은 원소 수가 $$n$$개인 입력 배열을 최악의 경우 $$\Theta(n^2)$$에 정렬하는 알고리즘이다. 퀵 정렬은 최악의 경우에는 수행시간이 느리지만 평균 수행시간이 $$\Theta(n \lg n)$$으로 매우 효율적이고 $$\Theta(n \lg n)$$ 표현에 숨겨진 상수 인자도 매우 적다. 또한 내부 정렬이라는 장점도 있고 가상 메모리 환경에서도 잘 동작해 일반적으로 실제 문제에서 가장 유용한 정렬 방법이다.

두 개의 부분 배열로 나누기 위한 중간 인덱스를 이 알고리즘에서는 **피봇**(pivot)이라고 부른다. 이 피봇을 기준으로 왼쪽에는 피봇보다 작은 값들을 배치하고 오른쪽에는 피봇보다 큰 값들을 배치한다.

![/assets/images/sort/28BBB9EA-F826-460D-B5D0-8D6046C9E475.gif](/assets/images/sort/28BBB9EA-F826-460D-B5D0-8D6046C9E475.gif)

빨간색(현재 인덱스), 노란색(피봇), 피봇보다 작은 값(초록색), 보라색(피봇보다 큰 값), 주황색(이미 정렬됨)

퀵 정렬은 병합 정렬과 마찬가지로 분할정복에 기반을 두고 있다. 다음은 배열 A[p..r]을 처리하는 3단계 분할정복 과정이다.

- 분할: 배열 A[p..r]을 두 개의 부분 배열 $$A[p..q-1]$$과 $$A[q+1..r]$$로 분할(재배치)한다. $$A[p..q-1]$$에는 $$A[q]$$보다 작거나 같은 원소를, $$A[q+1..r]$$에는 $$A[q]$$보다 크거나 같은 원소를 배치한다. 이 결과로 생기는 양쪽 부분 배열 중 한 쪽에는 아무 원소도 없을 수도 있다. 위의 분할 과정을 위한 인덱스 $$q$$를 계산한다.
- 정복: 퀵 정렬을 재귀 호출에서 $$A[p..q-1]$$과 $$A[q+1..r]$$ 두 부분 배열을 정렬한다.
- 결합: 부분 배열이 이미 정렬되어 있으므로 따로 합치는 작업을 할 필요가 없다. 전체 배열 $$A[p..r]$$이 이미 정렬되어 있기 때문이다.

### 의사코드

3-6행의 동작 과정은 아래와 같다.

![/assets/images/sort/E2669B65-F98F-4F70-B7AA-9A2252CCBB2C.jpeg](/assets/images/sort/E2669B65-F98F-4F70-B7AA-9A2252CCBB2C.jpeg)

(a) 배열과 변수의 초기 설정 상태: 맨 오른쪽 요소 4가 피봇으로 설정된다.

(b) 먼저 순회된 첫 번째 요소 2는 피봇 4보다 값이 작다. "자기 자신과 교환"하고 작은 값의 구역(옅은 회색)에 넣는다.

(c) 두 번째 요소 8은 피봇 4보다 값이 크므르 큰 값의 구역(짙은 회색)에 추가한다. 단순히 큰 값의 구역 인덱스를 1 추가한다.

(d) 세 번째 요소 7은 피봇 4보다 값이 크므로 큰 값의 구역에 추가한다.

(e) 네 번째 요소 1은 피봇 4보다 값이 작으므로 작은 값의 구역 오른쪽의 요소(8)와 자리를 바꾸고 작은 값의 구역 인덱스를 1 추가한다. 마찬가지로 큰 값의 구역 인덱스도 1 추가한다.

(f) 다섯 번째 요소 3은 피봇 4보다 값이 작으므로 작은 값의 구역 오른쪽의 요소(7)와 자리를 바꾸고 작은 값의 구역 인덱스를 1 추가한다. 마찬가지로 큰 값의 구역 인덱스도 1 추가한다.

(g) 여섯 번째 요소 5는 피봇 4보다 값이 크므로 큰 값의 구역에 추가한다.

(h) 일곱 번째 요소 6은 피봇 4보다 값이 크므로 큰 값의 구역에 추가한다.

(i) 다음 요소는 피봇이며, 루프의 종료 조건을 만족한다. 피봇을 작은 값의 구역과 큰 값의 구역 사이에 위치시키기 위해 큰 값의 구역 첫 번째 요소와 자리를 바꾼다.

### Python

```python
def quick_sort(A, p, r):
    if p < r:
        q = partition(A, p, r)
        quick_sort(A, p, q - 1)
        quick_sort(A, q + 1, r)

def partition(A, p, r):
    x = A[r - 1]
    i = p - 2
    for j in range(p - 1, r - 1):
        if A[j] <= x:
            i = i + 1
            A[i], A[j] = A[j], A[i]
    A[i + 1], A[r - 1] = A[r - 1], A[i + 1]
    return i + 2
```

## 성능 <a name="quick-sort-performance">

- 최악의 경우: $$O(n^2)$$
- 최선의 경우: $$O(n \lg n)$$

퀵 정렬의 수행시간은 분할 후 양쪽 구역의 크기가 비슷한지 그렇지 않은지에 따라 달라지고, 분할의 균형도는 분할할 때마다 어떤 기준 원소를 사용하는지에 달려 있다. 분할이 균등하게 되면 퀵 정렬은 (점근적으로) 병합 정렬처럼 빠르게 동작한다. 그러나 균등하지 않다면 삽입 정렬처럼 느리게 동작한다. 하지만 매번 배열 분할이 균등하게 이루어지지 않을 확률은 적을 뿐더러 내부적으로 수행되는 상수 $c$도 적기 때문에 평균적으로 좋은 성능을 낼 수 있다.

# 계수 정렬 (Counting Sort) <a name="counting-sort">

![/assets/images/sort/img.gif](/assets/images/sort/img.gif)

**계수 정렬**은 $$n$$개의 입력 원소 각각이 $$0$$부터 $$k$$ 사이에 있는 정수라고 가정한다. $$k=O(n)$$일 때 계수 정렬은 $$\Theta(n)$$ 시간에 수행된다.

계수 정렬은 각 입력 원소 $$x$$에 대해 $$x$$보다 작은 원소의 개수를 센다. 이 정보는 출력 배열에서 원소 $$x$$의 위치를 정하는 데 직접 사용된다. 예를 들어, $$x$$보다 작은 원소가 17개라면 $$x$$는 출력 배열에서 18번째 자리가 된다. 값이 같은 원소가 여러 개 있는 경우에는 모두 같은 자리에 둘 수 없으므로 방법을 약간 고쳐야 한다.

(a) 5행을 수행한 후 배열 $$A$$와 보조 배열 $$C$$

(b) 8행을 수행한 후의 배열 $$C$$

(c-e) 각각 10-12행을 한 번, 두 번, 세 번 반복한 후의 출력 배열 $$B$$와 보조 배열 $$C$$

(f) 최종 출력 배열 $$B$$

### Python

```python
def counting_sort(A, B, k):
    """
    B = [0] * len(A)
    k = max(A)
    """
    C = [0] * (k + 1)
    for j in range(len(A)):
        C[A[j]] += 1
    for i in range(1, k + 1):
        C[i] += C[i - 1]
    for j in reversed(range(len(A))):
        B[C[A[j]] - 1] = A[j]
        C[A[j]] -= 1
```

## 성능 <a name="counting-sort-performance">

총 네 개의 루프가 존재하는데, 첫 번째 루프는 배열 $$C$$를 0으로 초기화하는 것인데 파이썬에서는 반복문을 사용하지 않고 `[0] * (k + 1)` 로 표현할 수 있다. 아무튼 여기서는 $$\Theta(k)$$ 시간, 두 번째 루프에서는 $$\Theta(n)$$ 시간, 세 번째 루프에서는 $$\Theta(k)$$ 시간, 네 번째 루프에서는 $$\Theta(n)$$ 시간이 걸린다. 따라서 총 시간은 $$\Theta(k+n)$$이다. 실제 상황에서는 보통 $$k=O(n)$$일 때 계수 정렬을 사용하고, 이 경우 수행시간은 $$\Theta(n)$$이다. 퀵 정렬의 수행시간이 $$O(n \lg n)$$임을 떠올려보면 이는 아주 좋은 성능이지만, 정렬을 위해 원소의 개수를 세는 배열 $$C$$가 추가적으로 요구되므로 메모리 낭비가 심하다는 단점이 있다.

계수 정렬의 중요한 특징은 **안정성**(stable)을 가진다는 점이다. 이는 출력 배열에서 값이 같은 숫자가 입력 배열에 있던 것과 같은 순서로 나타나는 것을 뜻한다. 즉, 두 숫자가 같을 때는 입력 배열에서 먼저 나타나는 것이 출력 배열에서도 먼저 나타난다. 보통 안정성이라는 특성이 중요할 때는 정렬되는 원소에 부속 데이터가 붙어 다닐 때뿐이다. 계수 정렬의 안정성은 다른 이유로도 중요한데, 계수 정렬이 종종 기수 정렬의 서브 루틴으로 쓰이기 때문이다. 아래에서 설명될 기수 정렬이 정확하게 동작하기 위해서는 계수 정렬의 안정성이 필수이다.

# 기수 정렬 (Radix Sort) <a name="radix-sort">

**기수 정렬**은 이제는 컴퓨터 박물관에서나 볼 수 있는 카드 정렬 기계에서 사용하는 알고리즘이다. 낮은 자리수부터 비교하여 정렬하는 알고리즘이다.

![/assets/images/sort/radix-sort.gif](/assets/images/sort/radix-sort.gif)

## 구현 <a name="radix-sort-implementation">

### 의사코드

$n$개 원소의 배열 $A$에서 각 원소가 $d$자리로 되어 있고 1번 자리가 가장 낮은 자리며 $d$번 자리가 가장 높은 자리라고 가정한다.

![/assets/images/sort/8.3.png](/assets/images/sort/8.3.png)

### Python

```python
def counting_sort(A, d):
    B = [0] * len(A)
    C = [0] * 10
    for j in range(len(A)):
        digit = (A[j] // (10**d)) % 10
        C[digit] += 1
    for i in range(1, 10):
        C[i] += C[i - 1]
    for j in reversed(range(len(A))):
        digit = (A[j] // (10**d)) % 10
        B[C[digit] - 1] = A[j]
        C[digit] -= 1
    return B

def radix_sort(A, d):
    for i in range(d):
        A = counting_sort(A, i)
    return A
```

## 성능 <a name="radix-sort-performance">

흔한 경우로 $$b=O(\lg n)$$이고 $$r \approx \lg n$$ 을 선택하면 기수 정렬의 수행시간은 $$\Theta(n)$$이고 이것은 퀵 정렬의 평균 수행시간인 $$\Theta(n \lg n)$$보다 좋아 보인다. 그러나 $$\Theta$$-표현에 숨겨진 상수 인자는 다르다. 기수 정렬이 퀵 정렬에 비해서 키를 처리하는 과정 수가 적더라도 각각의 과정은 매우 길게 걸릴 수 있다. 어떤 정렬 알고리즘이 보다 나은지는 구현, 사용하는 기계(예를 들어, 퀵 정렬은 보통 기수 정렬에 비해 캐시를 효율적으로 사용한다), 입력 자료의 특성에 따라 다르다. 게다가 계수 정렬을 중간 정렬로 사용하는 기수 정렬은 $$\Theta(n \lg n)$$ 시간 걸리는 많은 비교 정렬들처럼 내부 정렬이 아니다. 따라서 메인 메모리가 귀한 상황에서는 퀵 정렬 같은 내부 정렬 알고리즘이 더 좋을 수 있다.

# 버킷 정렬 (Bucket Sort) <a name="bucket-sort">
