### 문제 설명

두 개의 단어 begin, target과 단어의 집합 words가 있습니다. 아래와 같은 규칙을 이용하여 begin에서 target으로 변환하는 가장 짧은 변환 과정을 찾으려고 합니다.

```
1. 한 번에 한 개의 알파벳만 바꿀 수 있습니다.
2. words에 있는 단어로만 변환할 수 있습니다.
```

예를 들어 begin이 "hit", target가 "cog", words가 ["hot","dot","dog","lot","log","cog"]라면 "hit" -> "hot" -> "dot" -> "dog" -> "cog"와 같이 4단계를 거쳐 변환할 수 있습니다.

두 개의 단어 begin, target과 단어의 집합 words가 매개변수로 주어질 때, 최소 몇 단계의 과정을 거쳐 begin을 target으로 변환할 수 있는지 return 하도록 solution 함수를 작성해주세요.


### 제한사항

- 각 단어는 알파벳 소문자로만 이루어져 있습니다.
- 각 단어의 길이는 3 이상 10 이하이며 모든 단어의 길이는 같습니다.
- words에는 3개 이상 50개 이하의 단어가 있으며 중복되는 단어는 없습니다.
- begin과 target은 같지 않습니다.
- 변환할 수 없는 경우에는 0를 return 합니다.


### 입출력 예

| begin | target | words                                      | return |
|-------|--------|--------------------------------------------|--------|
| "hit" | "cog"  | ["hot", "dot", "dog", "lot", "log", "cog"] | 4      |
| "hit" | "cog"  | ["hot", "dot", "dog", "lot", "log"]	      | 0      |

## 처음 문제 풀이 (DFS)

입출력 예1의 탐색 과정을 아래 그림처럼 표현할 수 있다. 발견된 target 중 가장 낮은 레벨은 4이다.

<img src="/assets/images/2021-04-20-programmers-43163/1.png" width="500px">

- `begin`으로 주어진 노드부터 시작해서 알파벳 하나만 차이 나는 노드(이하 후보 노드라고 명칭)들을 탐색한다.

- 입출력 예1처럼 `begin="hit"`, `target="cog"`, `words=["hot”, "dot”, "dog”, "lot”, "log”, "cog"]`로 입력이 주어진 경우, 처음에는 "hit"의 후보 노드를 탐색한다.

    후보 노드의 조건을 더 자세히 따져보면 다음과 같이 정리할 수 있다.
    - 이전 노드와 비교해 문자 하나만 변경되어야 함.
    - 이전에 방문한 적 없는 노드여야 함.

- `words`에서 후보 노드는 "hot"이 유일하다. 그러므로 "hit"에서는 "hot"으로만 변경할 수 있다.

- 이제 "hot"의 후보 노드를 탐색한다. 그 후보군은 `["dot, "lot"]`이다.

- 그 중 "dot"의 후보 노드 탐색 결과는 `["dog", "lot"]`이다.

- 그 중 "dog"의 후보 노드 탐색 결과는 `["log", "cog"]`이다.

- 후보 노드 탐색 결과에 `target`이 포함되어 있으므로, `target`의 수준을 `return` 한다.

- 이 입출력 예에는 `target`이 `words`에 포함되어 있으므로 언젠가는 0이 아닌 정수가 `return` 되겠지만,
  만약 그렇지 않은 경우 모든 노드를 탐색 후 0을 반환할 것이다.

- 그리고 위에서 후보 노드에 대한 탐색 함수를 호출한 부모 노드들 입장에서는 각기 다른 반환값과 함께 리턴된 자식 노드들을 가지고 그 중 가장 작은 값을 다시 자신의 부모에게 리턴하면 된다.

```python
from collections import Counter

def solution(begin, target, words):
    def find_candidates(node):
        """주어진 노드(node)와 한 문자만 차이나는 모든 단어를 찾는다."""
        candidates = set()
        for word in words:
            if word == node:
                continue
            if len(Counter(word) - Counter(node)) == 1:
                candidates.add(word)
        return candidates

    def dfs(node, level, visited):
        # 주어진 노드와 알파벳 하나만 차이 나는 모든 후보 노드(단어)를 가져오되,
        # 이미 방문한 적 있는 노드는 제외한다.
        visited.add(node)
        candidates = find_candidates(node) - visited

        # 종료 조건 1: 남은 후보가 없는 경우 더 이상 변환을 진행할 수 없으므로 0 리턴
        if len(candidates) == 0:
            return 0
        # 종료 조건 2: 후보에 target이 존재하는 경우 target의 레벨을 리턴
        elif target in candidates:
            return level + 1

        # 주어진 후보에 대해서 dfs()를 호출하고 리턴 값을 저장한다.
        target_levels = []
        for candidate in candidates:
            target_level = dfs(candidate, level + 1, visited.copy())
            target_levels.append(target_level)

        # 저장된 리턴 값 중 0이 아닌 가장 작은 값을 리턴한다.
        # 만약 값이 없다면 target을 찾지 못한 것이므로 0을 반환한다.
        answers = [x for x in target_levels if isinstance(x, int) and x != 0]
        return min(answers) if len(answers) > 0 else 0

    if target not in words:
        return 0

    return dfs(begin, 0, set())
```

### 아쉬운 점

재귀 함수를 사용하면 그 특성 상 DFS 알고리즘의 특성을 띠게 된다. 이런 식으로 깊이 우선 탐색을 시도하면 트리에서 `target`의 최소 레벨을 구하고 비교하기 위해 모든 노드를 탐색해야 한다는 문제가 있다. 예를 들어 처음 tree 그림 예시에서 "hot" 밑에 "dot"과 "lot"이 있는데 각각 `target`이 발견되는 최소 레벨을 구하기 위해 양쪽에서 모든 노드를 구한 뒤 서로를 비교해야 한다. 이런 방식 대신 너비 우선으로 탐색하는 BFS 알고리즘을 사용한다면 위 그림에서 레벨 1씩 높여가면서 해당 레벨에 있는 모든 노드를 우선으로 탐색할 것이다. 그리고 그 레벨에서 `target`이 발견되는 즉시 현재 레벨을 반환하도록 처리하면 DFS 알고리즘에 비해 성능 면에서 훨씬 효율적일 것이다.


## 개선 답안 (BFS)

BFS 알고리즘을 사용하면서 어려웠던 점 중 하나는 방문처리 배열(`visited`)의 관리였다.
DFS 알고리즘에서는 깊이 우선으로 탐색하면서 방문한 노드를 방문처리 배열에 순서대로 기록하면 되었기 때문에
방문처리 배열을 하나만 사용해도 문제가 없었다.
그러나 DFS 알고리즘에서는 너비 우선으로 노드를 방문하기 때문에 각각의 노드의 대한 방문처리 배열을 수시로 업데이트 해야 한다는 문제가 있었다.
그래서 노드 별로 방문처리 목록을 (`visit_history`) 만들어 관리하기 위해 `Node` 클래스를 정의하여 사용했다.

```python
from collections import Counter, deque


class Node:
    def __init__(self, word, visit_history):
        self.word = word
        self.visit_history = visit_history

    def __repr__(self):
        if self.visit_history:
            return " -> ".join(self.visit_history) + " -> " + self.word
        else:
            return self.word


def find_candidate_words(node, words):
    candidate_words = []
    for word in words:
        if word == node.word:
            continue
        if word in node.visit_history:
            continue
        if len(Counter(word) - Counter(node.word)) != 1:
            continue
        candidate_words.append(word)
    return candidate_words


def solution(begin, target, words):
    if target not in words:
        return 0

    root_node = Node(word=begin, visit_history=[])
    queue = deque([root_node])

    while queue:
        node = queue.popleft()

        # 탐색된 노드의 단어가 target인 경우 해당 노드의 레벨 반환
        if node.word == target:
            return len(node.visit_history)

        # 다음 레벨의 노드를 큐에 추가
        for word in find_candidate_words(node, words):
            candidate_node = Node(word, node.visit_history + [node.word])
            queue.append(candidate_node)
```
